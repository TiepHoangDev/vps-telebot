# VPS Telebot

A Telegram bot for managing your VPS and Docker Compose projects — directly from your phone.

## Features

- **Docker Compose management** — add any `docker-compose.yml` path and get instant commands: up, down, logs, pull, deploy
- **Custom commands** — add any shell command as a bot command under a project
- **VPS monitoring** — CPU, RAM, disk, uptime, load average, OS info
- **GitHub Actions deploy** — trigger deploys from CI/CD using a project secret (no open ports needed)
- **Auth** — password-protected setup, user allowlist

## Prerequisites

- A VPS running Linux
- Docker + Docker Compose installed on the VPS
- A Telegram bot token from [@BotFather](https://t.me/BotFather)

## Deploy

### 1. Prepare files on your VPS

```bash
mkdir vps-telebot && cd vps-telebot
```

Create `docker-compose.yml`:

```yaml
services:
  telebot:
    image: ghcr.io/YOUR_GITHUB_USERNAME/vps-telebot:latest
    restart: unless-stopped
    volumes:
      - ./data.json:/app/data.json
      - /var/run/docker.sock:/var/run/docker.sock
    env_file:
      - .env
```

### 2. Configure environment

Create `.env` — **never commit this file**:

```env
BOT_TOKEN=your_telegram_bot_token
SETUP_PASS=a_strong_setup_password
DATA_PATH=/app/data.json
```

### 3. Run

```bash
touch data.json
docker compose up -d
```

### 4. First login

Open a private chat with your bot and send your `SETUP_PASS`. You are now the owner.

## Bot Commands

| Command | Description |
|---|---|
| `/help` | Show all commands and projects |
| `/addproject` | Add a docker-compose.yml path |
| `/addcmd` | Add a custom shell command |
| `/delcmd` | Delete a command |
| `/vps` | Show VPS resource usage |
| `/deploy <project> <secret>` | Trigger deploy (used by GitHub Actions) |

Once you add a project (e.g. `myapp`), these commands are auto-created:

| Command | Action |
|---|---|
| `/myapp_up` | `docker compose up -d` |
| `/myapp_down` | `docker compose down` |
| `/myapp_downv` | `docker compose down -v` |
| `/myapp_logs` | `docker compose logs --tail=50` |
| `/myapp_pull` | `docker compose pull` |
| `/myapp_deploy` | pull + up -d |

Click a project in `/help` to manage it: add commands, set deploy secret, or delete.

## GitHub Actions Auto-Deploy

This lets GitHub trigger a deploy on push — without exposing any ports on your VPS.

### How it works

1. In the bot, click your project → **🔑 Deploy Secret** → copy the secret
2. Add these secrets to your GitHub repo (`Settings → Secrets → Actions`):

| Secret | Value |
|---|---|
| `BOT_TOKEN` | Your Telegram bot token |
| `CHAT_ID` | Your Telegram user ID (get it from [@userinfobot](https://t.me/userinfobot)) |
| `DEPLOY_SECRET` | The secret generated by the bot |

3. Add this step to your workflow:

```yaml
- name: Trigger deploy
  run: |
    curl -s "https://api.telegram.org/bot${{ secrets.BOT_TOKEN }}/sendMessage" \
      -d "chat_id=${{ secrets.CHAT_ID }}" \
      -d "text=/deploy myapp ${{ secrets.DEPLOY_SECRET }}"
```

The bot verifies the secret, runs the deploy command, and sends you the output.

> **Note:** Use a private chat with the bot (not a group) — the `/deploy` message will be visible in chat history.

## Build from Source

```bash
git clone https://github.com/YOUR_USERNAME/vps-telebot.git
cd vps-telebot
npm install
cp .env.example .env   # fill in BOT_TOKEN and SETUP_PASS
npm run dev
```

### CI/CD — Automatic Docker builds

Push to `main` and the GitHub Actions workflow (`.github/workflows/build.yml`) will automatically build and push the Docker image to `ghcr.io/YOUR_USERNAME/vps-telebot:latest`.

**Setup:**
1. Fork or push to your own GitHub repo
2. Go to `Settings → Packages` and make the package public (or keep it private)
3. Update `docker-compose.yml` on your VPS with your image URL

## Security Notes

- `.env` and `data.json` are in `.gitignore` — never commit them
- Bot only responds to users in the `allowed_users` list
- Deploy endpoint (`/deploy`) validates a per-project secret, not user identity — keep `DEPLOY_SECRET` in GitHub Secrets, not in plaintext
- Mount `/var/run/docker.sock` gives the bot full Docker access — run on a trusted VPS only
- `SETUP_PASS` is only needed for first-time setup; once the owner is registered it has no effect

## Data Storage

All configuration is stored in `data.json` (mounted as a volume). Structure:

```json
{
  "allowed_users": [123456789],
  "projects": {
    "myapp": {
      "path": "/root/myapp/docker-compose.yml",
      "deploy_secret": "xxxx-xxxx-xxxx",
      "commands": {
        "myapp_up": "docker compose -f /root/myapp/docker-compose.yml up -d"
      }
    }
  }
}
```
