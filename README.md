# VPS Telebot

A Telegram bot for managing your VPS and Docker Compose projects — directly from your phone. Button-first UI, no commands to memorise.

## Features

- **Button-driven UI** — everything accessible via inline buttons, no need to type commands.
- **Docker Compose management** — add any `docker-compose.yml` and get instant buttons: up, down, logs, pull, deploy.
- **Custom commands** — add any shell command as a button under a project.
- **VPS monitoring** — CPU, RAM, disk, uptime, load average, OS info.
- **GitHub Actions deploy** — trigger deploys from CI/CD via a project secret (no open ports needed).
- **Auth** — password-protected setup, user allowlist.

## Prerequisites

- A VPS running Linux
- Docker + Docker Compose installed on the VPS
- A Telegram bot token from [@BotFather](https://t.me/BotFather)

## Deploy

### 1. Prepare files on your VPS

```bash
mkdir vps-telebot && cd vps-telebot
```

Copy [`docker-compose.yml`](docker-compose.yml) from this repo.

### 2. Configure environment

Copy [`.env.example`](.env.example) to `.env` and fill in the values:

| Variable | Description |
|---|---|
| `BOT_TOKEN` | Telegram bot token from @BotFather |
| `SETUP_PASS` | Password for first-time owner registration |
| `DATA_PATH` | Path to data file (default: `/app/data/data.json`) |

### 3. Run

```bash
docker compose up -d
```

### 4. First login

Open a **private chat** with your bot and send your `SETUP_PASS`. You become the owner and the bot is ready to use.

## Usage

### `/help`

The main entry point. Shows the current version and two action buttons, plus all configured projects.

```
VPS Telebot  v1.x.x

[ ➕ Add Project ]
[ 📊 VPS Status  ]

📋 Projects — click to manage:
[ 📁 myapp ]
```

### Managing a project

Click any project button to open its view:

```
📁 myapp
Path: /root/myapp/docker-compose.yml

[ ▶ up      ] [ ⏹ down   ] [ ⏹ down -v ]
[ 📋 logs   ] [ ⬇ pull   ] [ 🚀 deploy  ]
[ my_custom ]

[ ➕ Add Cmd ] [ 🗑 Del Cmd ]
[ 📤 Send File               ]
[ 🔑 Deploy Secret          ]
[ 🗑 Delete Project          ]
[ « Back                    ]
```

Clicking a command button executes it immediately and shows the output.

### Adding a project

Click **➕ Add Project**, send the path to your `docker-compose.yml`:

```
/root/myapp/docker-compose.yml
```

The bot derives the project name from the folder (`myapp`) and auto-creates 6 buttons:

| Button | Action |
|---|---|
| `▶ up` | `docker compose up -d` |
| `⏹ down` | `docker compose down` |
| `⏹ down -v` | `docker compose down -v` |
| `📋 logs` | `docker compose logs --tail=50` |
| `⬇ pull` | `docker compose pull` |
| `🚀 deploy` | pull + up -d |

### Adding a custom command

Open a project → **➕ Add Cmd**. Enter a suffix (e.g. `restart_worker`) — the bot saves it as `myapp_restart_worker` and adds it as a button to the project view.

### Deleting a command

Open a project → **🗑 Del Cmd** → pick the command to delete.

## GitHub Actions Auto-Deploy

Trigger deploys on push without exposing any ports on your VPS.

### Setup

1. Open a project in the bot → **🔑 Deploy Secret** — the bot generates a secret and shows a ready-made GitHub Actions snippet.

2. Add these to your GitHub repo (`Settings → Secrets → Actions`):

| Secret | Value |
|---|---|
| `BOT_TOKEN` | Your Telegram bot token |
| `CHAT_ID` | Your Telegram user ID — get it from [@userinfobot](https://t.me/userinfobot) |
| `DEPLOY_SECRET` | The secret generated by the bot |

3. Add to your workflow:

```yaml
- name: Trigger deploy
  run: |
    curl -s "https://api.telegram.org/bot${{ secrets.BOT_TOKEN }}/sendMessage" \
      -d "chat_id=${{ secrets.CHAT_ID }}" \
      -d "text=/deploy myapp ${{ secrets.DEPLOY_SECRET }}"
```

The bot verifies the secret, runs `myapp_deploy`, and sends the output back to you on Telegram.

> **Note:** Use a private 1-on-1 chat with the bot — the `/deploy` message (containing the secret) will be visible in chat history.

## Build from Source

```bash
git clone https://github.com/tiephoangdev/vps-telebot.git
cd vps-telebot
npm install
cp .env.example .env   # fill in BOT_TOKEN and SETUP_PASS
npm run dev
```

### CI/CD — Automatic Docker image builds

Every push to `master` triggers `.github/workflows/build.yml`, which builds and pushes the image to `ghcr.io/tiephoangdev/vps-telebot:latest`. Releases are also tagged with the semver version.

## Security Notes

- `.env` and `data.json` are in `.gitignore` — never commit them
- Bot ignores all messages from users not in `allowed_users`
- `/deploy` is validated by a per-project secret, not by user identity — store `DEPLOY_SECRET` in GitHub Secrets only
- Mounting `/var/run/docker.sock` gives the bot full Docker access — run on a trusted server only
- `SETUP_PASS` only has effect when `allowed_users` is empty; once an owner is registered it does nothing

## Data Storage

All state is stored in `data.json` (mounted as a volume). You can back it up or edit it manually.

```json
{
  "allowed_users": [123456789],
  "projects": {
    "myapp": {
      "path": "/root/myapp/docker-compose.yml",
      "deploy_secret": "xxxx-xxxx-xxxx",
      "commands": {
        "myapp_up":     "docker compose -f /root/myapp/docker-compose.yml up -d",
        "myapp_deploy": "docker compose -f /root/myapp/docker-compose.yml pull && docker compose -f /root/myapp/docker-compose.yml up -d",
        "myapp_restart_worker": "docker compose -f /root/myapp/docker-compose.yml restart worker"
      }
    }
  }
}
```
